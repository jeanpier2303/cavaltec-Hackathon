Arquitectura del proyecto

El proyecto se organiza en dos aplicaciones independientes —backend y frontend— más el presente archivo de documentación en la raíz.

backend/
Aplicación Python con FastAPI, SQLAlchemy y Docker
frontend/
Aplicación React con Vite y React Router
documentacion.html
Documentación del proyecto (este archivo)

Backend

API REST construida con FastAPI. Expone endpoints para la gestión de evaluaciones, usuarios, empresas y reportes. Se conecta a MySQL mediante SQLAlchemy y administra migraciones con Alembic. Se despliega con Docker Compose junto a MySQL y phpMyAdmin.

Frontend

SPA desarrollada en React con Vite. Proporciona una interfaz moderna con tres layouts según el rol del usuario (Super Administrador, Administrador de Empresa, Auditor). Incluye panel de IA, dashboards con métricas y flujo completo de evaluación.

Requisitos previos

Antes de comenzar, asegúrate de tener instalado lo siguiente:

Git

Control de versiones. Descárgalo desde git-scm.com.

Docker Desktop

Entorno de contenedores para el backend y la base de datos. Descárgalo desde docker.com.

Node.js + npm

Entorno de ejecución para el frontend. Descárgalo desde nodejs.org.

5

Clonar el proyecto

Abre una terminal y ejecuta los siguientes comandos: 

# Clonar el repositorio
git clone https://github.com/usuario/cavaltec.git

# Entrar al directorio del proyecto
cd cavaltec
Reemplaza la URL del repositorio con la correcta cuando esté disponible.
6

Ejecutar el Backend

1

Entrar a la carpeta del backend

cd backend
2

Levantar los servicios con Docker

docker compose up --build

Este comando iniciará tres servicios:

  • Backend FastAPI — API REST en localhost:8000
  • MySQL 8 — Base de datos relacional
  • phpMyAdmin — Administrador visual de BD en localhost:8080
Asegúrate de que Docker Desktop esté corriendo antes de ejecutar docker compose up.
7

URLs del Backend

Una vez que Docker esté funcionando, los siguientes servicios estarán disponibles:

Swagger UI http://localhost:8000/docs
Health Check http://localhost:8000/health
API Base http://localhost:8000
phpMyAdmin http://localhost:8080
Los puertos pueden variar según la configuración del archivo docker-compose.yml.
8

Ejecutar el Frontend

1

Entrar a la carpeta del frontend

cd frontend
2

Instalar dependencias

npm install
3

Iniciar el servidor de desarrollo

npm run dev

La aplicación estará disponible en:

http://localhost:5173

Flujo de demostración

Sigue estos pasos para probar la aplicación en modo demostración:

1

Abrir la Landing

Página principal con información del producto, características y llamado a la acción.

2

Ir al Login

Haz clic en "Iniciar sesión" desde la Landing o navega a /login.

3

Seleccionar un rol

En la sección "Modo Demostración", elige entre Super Administrador, Administrador de Empresa o Auditor.

4

Iniciar sesión

Ingresa cualquier correo y contraseña. El sistema redirige al dashboard correspondiente al rol seleccionado.

5

Recorrer las pantallas

Explora las diferentes secciones: dashboard, evaluaciones, historial, reportes, panel IA y configuración. Navega entre las vistas usando el menú lateral.

Actualmente la aplicación funciona con datos estáticos de demostración. La integración con el backend estará disponible próximamente.
10

Roles del sistema

SA

Super Administrador

Acceso global a la plataforma. Gestiona empresas, usuarios, cuestionarios, versiones, reportes y auditoría. Visualiza dashboards consolidados con métricas de todas las empresas.

AE

Administrador de Empresa

Gestiona los usuarios y evaluaciones de su empresa. Visualiza dashboards con métricas propias, ejecuta evaluaciones y genera reportes. Accede al panel IA para recomendaciones.

AU

Auditor

Revisa evaluaciones asignadas, genera reportes de auditoría y consulta el historial. Visualiza dashboards con el cumplimiento promedio de las empresas que audita.
